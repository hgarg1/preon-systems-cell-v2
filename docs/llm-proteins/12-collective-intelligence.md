# §12 — Collective Intelligence Formation

## Core Principle
> Many components think, but **only one system decides**.

This layer is the transition from distributed computation → unified cognition, parallel execution → coherent reasoning, fragmented outputs → singular decisions.

## Two-Direction Flow

### Bottom-Up (Emergence)
```
Proteins → atomic tasks
Cells    → coordinate + aggregate
Tissues  → unify functional reasoning
Organs   → synthesise domain-level intelligence
Organism → final decision
```

### Top-Down (Control)
```
Organism → sets objectives
Organs   → direct tissues
Tissues  → influence cell execution
Cells    → guide protein behaviour
```

Both flows operate **simultaneously**.

## Multi-Level Aggregation

| Transition | What happens |
|---|---|
| Protein → Cell | Atomic results → intermediate reasoning |
| Cell → Tissue | Intermediate outputs → validated results |
| Tissue → Organ | Validated results → domain intelligence |
| Organ → Organism | Domain intelligence → final decision |

Each transformation increases: abstraction, coherence, decision quality.

## Voice Unification
At every level, multiple contributors exist and outputs may conflict.  
The system must: reconcile differences · synthesise consistent outputs · present a single unified voice.

## 4 Consensus Mechanisms

| Mechanism | How |
|---|---|
| **Confidence-Weighted Aggregation** | Outputs weighted by confidence scores |
| **Hierarchical Override** | Higher-level structures override lower-level disagreements |
| **Evaluation-Based Selection** | Specialised evaluators score and select best outputs |
| **Iterative Convergence** | Outputs refined through repeated cycles until convergence |

## Cross-Organ Coordination
Multiple organs contributing to complex decisions must be: aligned · reconciled · merged into unified decision.

Example:
- Reasoning organ → generates plan
- Simulation organ → evaluates outcomes
- Execution organ → prepares actions

## Decision Formation
Final organism output based on: aggregated organ outputs · system objectives · constraints (latency, cost, risk).  
May include: structured results · reasoning summaries · action plans · confidence metrics.

## Feedback & Adaptation
Post-decision loops:
- Evaluate output quality
- Monitor downstream effects
- Re-invoke organs for refinement

Enables continuous improvement and adaptive behaviour.

## Consistency Guarantees
- Unified output schemas
- Hierarchical validation
- Controlled aggregation

Prevents: conflicting outputs · fragmented reasoning · incoherent decisions.

## Latency vs Accuracy Tradeoffs
- More aggregation → higher accuracy
- More coordination → higher latency

System dynamically balances speed requirements vs decision quality.

## Failure Handling
- Inability to reach consensus → fallback decision logic
- Conflicting organ outputs → partial output generation
- Incomplete reasoning chains → escalation or re-execution

## Example Flow
1. Multiple organs produce domain-specific outputs
2. Outputs aggregated at organism level
3. Conflicts identified and resolved
4. Unified decision synthesised
5. Feedback loops refine if necessary
6. Final result emitted

## Key Insight
This layer completes the transformation from **atomic LLM calls** → **organism-level intelligence**.
